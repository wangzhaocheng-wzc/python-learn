关系数据库概述

核心概念

数据持久化

将数据保存到能够长久保存数据的存储介质中，在掉电的情况下数据也不会丢失。

数据库发展史

网状数据库、层次数据库、关系数据库、NoSQL数据库、NewSQL数据库。

历史里程碑

1970年

IBM的研究员E.F.Codd在Communication of the ACM上发表了名为A Relational Model of Data for Large Shared Data Banks的论文，提出了关系模型的概念，奠定了关系模型的理论基础。后来Codd又陆续发表多篇文章，论述了范式理论和衡量关系系统的12条标准，用数学理论奠定了关系数据库的基础。

关系数据库特点

理论基础

集合论和关系代数

具体表象

用二维表（有行和列）组织数据

编程语言

结构化查询语言（SQL）

ER模型（实体关系模型）

ER模型，全称为实体关系模型（Entity-Relationship Model），由美籍华裔计算机科学家陈品山先生提出，是概念数据模型的高层描述方式。

ER模型组成要素

实体 - 矩形框
属性 - 椭圆框
关系 - 菱形框

重数关系

1:1 - 一对一关系
1:N - 一对多关系
M:N - 多对多关系

实际应用

实际项目开发中，我们可以利用数据库建模工具（如：PowerDesigner）来绘制概念数据模型（其本质就是ER模型），然后再设置好目标数据库系统，将概念模型转换成物理模型，最终生成创建二维表的SQL（很多工具都可以根据我们设计的物理模型图以及设定的目标数据库来导出SQL或直接生成数据表）。

范式理论

范式理论是设计关系型数据库中二维表的指导思想。

1

第一范式（1NF）

数据表的每个列的值域都是由原子值组成的，不能够再分割。

2

第二范式（2NF）

数据表里的所有数据都要和该数据表的键（主键与候选键）有完全依赖关系。

3

第三范式（3NF）

所有非键属性都只和候选键有相关性，也就是说非键属性之间应该是独立无关的。

数据完整性

实体完整性

每个实体都是独一无二的

主键（primary key）
唯一约束
唯一索引（unique）

引用完整性

关系中不允许引用不存在的实体

外键（foreign key）

域完整性

数据是有效的

数据类型及长度
非空约束（not null）
默认值约束（default）
检查约束（check）

在MySQL数据库中，检查约束并不起作用。

DDL（数据定义语言）

创建数据库和表结构

DDL用于定义数据库结构，包括创建、修改和删除数据库对象。

创建数据库 

-- 如果存在名为school的数据库就删除它
drop database if exists `school`;

-- 创建名为school的数据库并设置默认的字符集和排序方式
create database `school` default character set utf8mb4;

-- 切换到school数据库上下文环境
use `school`;

创建学院表 

-- 创建学院表
create table `tb_college`
(
`col_id` int unsigned auto_increment comment '编号',
`col_name` varchar(50) not null comment '名称',
`col_intro` varchar(5000) default '' comment '介绍',
primary key (`col_id`)
) engine=innodb comment '学院表';

创建学生表 

-- 创建学生表
create table `tb_student`
(
`stu_id` int unsigned not null comment '学号',
`stu_name` varchar(20) not null comment '姓名',
`stu_sex` boolean default 1 comment '性别',
`stu_birth` date not null comment '出生日期',
`stu_addr` varchar(255) default '' comment '籍贯',
`col_id` int unsigned not null comment '所属学院',
primary key (`stu_id`),
foreign key (`col_id`) references `tb_college` (`col_id`)
) engine=innodb comment '学生表';

创建教师表和课程表 

-- 创建教师表
create table `tb_teacher`
(
`tea_id` int unsigned not null comment '工号',
`tea_name` varchar(20) not null comment '姓名',
`tea_title` varchar(10) default '助教' comment '职称',
`col_id` int unsigned not null comment '所属学院',
primary key (`tea_id`),
foreign key (`col_id`) references `tb_college` (`col_id`)
) engine=innodb comment '老师表';

-- 创建课程表
create table `tb_course`
(
`cou_id` int unsigned not null comment '编号',
`cou_name` varchar(50) not null comment '名称',
`cou_credit` int unsigned not null comment '学分',
`tea_id` int unsigned not null comment '授课老师',
primary key (`cou_id`),
foreign key (`tea_id`) references `tb_teacher` (`tea_id`)
) engine=innodb comment '课程表';

重要说明

字符集设置： 创建数据库时，通过 default charset utf8mb4 指定了数据库默认使用的字符集，推荐使用该字符集，因为utf8mb4能够支持国际化编码，包括Emoji等图片字符。

查看字符集： 查看MySQL支持的字符集可以执行：show character set;

DQL（数据查询语言）

DQL是SQL中最常用的部分，主要用于数据查询，包括基本查询、条件查询、分组查询、排序等功能。

基本查询

-- 查询所有学生信息
select * from tb_student;

-- 查询指定列（投影）
select stu_name as 姓名, stu_birth as 生日 from tb_student;

条件查询

-- 查询所有女学生
select stu_name, stu_birth from tb_student where stu_sex=0;

-- 查询80后学生
select stu_name, stu_birth from tb_student 
where stu_birth between '1980-1-1' and '1989-12-31';

模糊查询

通配符

  • % - 匹配零个或多个字符
  • _ - 匹配一个字符
-- 查询姓"杨"的学生
select stu_name from tb_student where stu_name like '杨%';

-- 查询姓"杨"且名字两个字的学生
select stu_name from tb_student where stu_name like '杨_';

排序和分组

-- 按年龄排序
select stu_name, stu_birth from tb_student order by stu_birth desc;

-- 按性别分组统计
select if(stu_sex, '男', '女') as 性别, count(*) as 人数 
from tb_student group by stu_sex;

聚合函数

  • count() - 计数
  • sum() - 求和
  • avg() - 平均值
  • max() - 最大值
  • min() - 最小值
-- 计算平均成绩
select avg(score) from tb_record;

-- 统计学生人数
select count(*) from tb_student;

连接查询

连接类型

  • 内连接（INNER JOIN）
  • 左外连接（LEFT JOIN）
  • 右外连接（RIGHT JOIN）
  • 全外连接（在MySQL中通过UNION实现）
-- 查询学生和所属学院
select stu.stu_name, col.col_name
from tb_student stu
inner join tb_college col on stu.col_id = col.col_id;

-- 查询所有学生的选课情况（包括没有选课的学生）
select stu.stu_name, cou.cou_name, rec.score
from tb_student stu
left join tb_record rec on stu.stu_id = rec.sid
left join tb_course cou on rec.cid = cou.cou_id;

子查询

子查询类型

  • 标量子查询（返回单个值）
  • 列子查询（返回一列值）
  • 行子查询（返回一行值）
  • 表子查询（返回多行多列）
-- 查询成绩高于平均分的学生
select stu_name, score
from tb_student stu
inner join tb_record rec on stu.stu_id = rec.sid
where score > (select avg(score) from tb_record);

-- 查询选修了Python课程的学生信息
select stu_name, stu_birth
from tb_student
where stu_id in (
    select sid from tb_record
    where cid = (
        select cou_id from tb_course
        where cou_name = 'Python程序设计'
    )
);

高级查询技巧

-- 使用窗口函数计算每个学生的成绩排名
select 
    stu.stu_name,
    cou.cou_name,
    rec.score,
    rank() over (partition by rec.cid order by rec.score desc) as rank
from tb_student stu
inner join tb_record rec on stu.stu_id = rec.sid
inner join tb_course cou on rec.cid = cou.cou_id;

-- 使用WITH子句（CTE）简化复杂查询
with avg_scores as (
    select sid, avg(score) as avg_score
    from tb_record
    group by sid
)
select stu.stu_name, avg_scores.avg_score
from tb_student stu
inner join avg_scores on stu.stu_id = avg_scores.sid
where avg_scores.avg_score >= 85;

索引

索引是关系型数据库中用来提升查询性能最为重要的手段。关系型数据库中的索引就像一本书的目录，我们可以想象一下，如果要从一本书中找出某个知识点，但是这本书没有目录，这将是意见多么可怕的事情（我们估计得一篇一篇的翻下去，才能确定这个知识点到底在什么位置）。

创建索引虽然会带来存储空间上的开销，就像一本书的目录会占用一部分的篇幅一样，但是在牺牲空间后换来的查询时间的减少也是非常显著的。

MySQL索引特性

所有数据类型的列都可以被索引
InnoDB和MyISAM支持每个表创建16个索引
底层算法使用B-tree（B树）

B-tree特点

自平衡的树结构
能够保持数据有序
查找、插入、删除都在对数时间内完成

数据一致性与事务

事务概念

事务：一系列对数据库进行读/写的操作，这些操作要么全都成功，要么全都失败。

事务的ACID特性

A

原子性 (Atomicity)

事务作为一个整体被执行，包含在其中的对数据库的操作要么全部被执行，要么都不执行

C

一致性 (Consistency)

事务应确保数据库的状态从一个一致状态转变为另一个一致状态

I

隔离性 (Isolation)

多个事务并发执行时，一个事务的执行不应影响其他事务的执行

D

持久性 (Durability)

已被提交的事务对数据库的修改应该永久保存在数据库中

批量插入

def batch_insert(conn, table_name, data_list):
    with conn.cursor() as cursor:
        # 构建SQL语句
        placeholders = ', '.join(['(%s, %s, %s)'] * len(data_list))
        sql = f'INSERT INTO {table_name} (name, age, score) VALUES {placeholders}'
        
        # 展平数据列表
        flat_data = []
        for item in data_list:
            flat_data.extend([item['name'], item['age'], item['score']])
        
        # 执行批量插入
        cursor.execute(sql, flat_data)
    conn.commit()

# 使用示例
data = [

    {'name': '李四', 'age': 22, 'score': 90},
    {'name': '王五', 'age': 21, 'score': 88}
]
batch_insert(conn, 'students', data)

事务处理

def transfer_money(conn, from_account, to_account, amount):
    try:
        with conn.cursor() as cursor:
            # 检查余额
            cursor.execute(
                'SELECT balance FROM accounts WHERE id = %s FOR UPDATE',
                (from_account,)
            )
            from_balance = cursor.fetchone()[0]
            
            if from_balance < amount:
                raise Exception('余额不足')
            
            # 扣款和存款
            cursor.execute(
                'UPDATE accounts SET balance = balance - %s WHERE id = %s',
                (amount, from_account)
            )
            cursor.execute(
                'UPDATE accounts SET balance = balance + %s WHERE id = %s',
                (amount, to_account)
            )
            
        conn.commit()
        return True
    except Exception as e:
        conn.rollback()
        print(f'转账失败：{str(e)}')
        return False

ORM风格封装

class BaseDAO:
    def __init__(self, pool):
        self.pool = pool
    
    def execute(self, sql, args=None):
        conn = self.pool.connection()
        try:
            with conn.cursor() as cursor:
                cursor.execute(sql, args)
                conn.commit()
                return cursor.rowcount
        finally:
            conn.close()
    
    def query_one(self, sql, args=None):
        conn = self.pool.connection()
        try:
            with conn.cursor(pymysql.cursors.DictCursor) as cursor:
                cursor.execute(sql, args)
                return cursor.fetchone()
        finally:
            conn.close()
    
    def query_all(self, sql, args=None):
        conn = self.pool.connection()
        try:
            with conn.cursor(pymysql.cursors.DictCursor) as cursor:
                cursor.execute(sql, args)
                return cursor.fetchall()
        finally:
            conn.close()

# 使用示例
class StudentDAO(BaseDAO):
    def get_by_id(self, student_id):
        return self.query_one(
            'SELECT * FROM students WHERE id = %s',
            (student_id,)
        )
    
    def get_all_by_score(self, min_score):
        return self.query_all(
            'SELECT * FROM students WHERE score >= %s ORDER BY score DESC',
            (min_score,)
        )
    
    def update_score(self, student_id, new_score):
        return self.execute(
            'UPDATE students SET score = %s WHERE id = %s',
            (new_score, student_id)
        )

避免使用root账户进行日常操作。

窗口函数

窗口函数（也称为分析函数）是MySQL 8.0版本引入的新特性，可以在查询中对数据进行分组计算，同时保留原始行的详细信息。

基本语法

function_name() OVER (
    [PARTITION BY column_list]
    [ORDER BY column_list]
    [frame_clause]
)

PARTITION BY：指定分组的列

ORDER BY：指定排序方式

frame_clause：定义窗口范围

常用窗口函数

排序函数

ROW_NUMBER() 连续排名
RANK() 跳跃排名
DENSE_RANK() 连续密集排名

分析函数

LAG() 获取前面的值
LEAD() 获取后面的值
FIRST_VALUE() 第一个值

实际应用示例

-- 按部门分组，计算员工工资排名
SELECT 
    emp_name,
    department,
    salary,
    RANK() OVER (
        PARTITION BY department 
        ORDER BY salary DESC
    ) as salary_rank
FROM employees;

-- 计算销售额的同比增长
SELECT 
    month,
    sales,
    LAG(sales, 12) OVER (ORDER BY month) as last_year_sales,
    (sales - LAG(sales, 12) OVER (ORDER BY month)) / 
    LAG(sales, 12) OVER (ORDER BY month) * 100 as growth_rate
FROM monthly_sales;

高级查询

高级查询技巧包括子查询、连接查询、聚合函数、窗口函数等。

子查询

-- 查询成绩高于平均分的学生
select stu_name, score
from tb_student stu
inner join tb_record rec on stu.stu_id = rec.sid
where score > (select avg(score) from tb_record);

-- 查询选修了Python课程的学生信息
select stu_name, stu_birth
from tb_student
where stu_id in (
    select sid from tb_record
    where cid = (
        select cou_id from tb_course
        where cou_name = 'Python程序设计'
    )
);

连接查询

-- 查询学生和所属学院
select stu.stu_name, col.col_name
from tb_student stu
inner join tb_college col on stu.col_id = col.col_id;

-- 查询所有学生的选课情况（包括没有选课的学生）
select stu.stu_name, cou.cou_name, rec.score
from tb_student stu
left join tb_record rec on stu.stu_id = rec.sid
left join tb_course cou on rec.cid = cou.cou_id;

聚合函数

-- 计算平均成绩
select avg(score) from tb_record;

-- 统计学生人数
select count(*) from tb_student;

窗口函数

-- 使用窗口函数计算每个学生的成绩排名
select 
    stu.stu_name,
    cou.cou_name,
    rec.score,
    rank() over (partition by rec.cid order by rec.score desc) as rank
from tb_student stu
inner join tb_record rec on stu.stu_id = rec.sid
inner join tb_course cou on rec.cid = cou.cou_id;

-- 使用WITH子句（CTE）简化复杂查询
with avg_scores as (
    select sid, avg(score) as avg_score
    from tb_record
    group by sid
)
select stu.stu_name, avg_scores.avg_score
from tb_student stu
inner join avg_scores on stu.stu_id = avg_scores.sid
where avg_scores.avg_score >= 85;
